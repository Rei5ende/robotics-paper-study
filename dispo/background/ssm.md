# 📚 Background Study: State Space Model (SSM) & Mamba

> DiSPo의 핵심 아키텍처인 Mamba를 이해하기 위한 배경 공부 노트  
> 관련 논문: DiSPo (ICRA 2026)

---

## 1. SSM이 왜 필요한가 — 트랜스포머의 한계부터

딥러닝에서 시계열 데이터(순서가 있는 데이터)를 처리하는 주류 방법은 **트랜스포머**다.  
그런데 트랜스포머에는 로봇 제어에서 치명적인 한계가 있다.

### 트랜스포머의 문제: Temporal Structure를 모른다

트랜스포머는 시퀀스의 모든 위치를 동등하게 본다.

```
시퀀스: [t=0, t=1, t=2, t=3, t=4]
트랜스포머: 모든 위치를 동시에 참조 (Attention)
            → "t=0이 t=3보다 0.3초 앞이다"는 개념이 없음
            → 그냥 위치 인덱스 0, 1, 2, 3, 4로만 구분
```

로봇 제어에서는 **시간의 연속성**이 매우 중요하다.
- 0.1초 간격으로 샘플링한 데이터 vs 1초 간격으로 샘플링한 데이터
- 트랜스포머 입장에서는 둘 다 그냥 "길이 10짜리 시퀀스"

---

## 2. SSM이란? — 제어공학에서 온 개념

SSM은 원래 제어공학에서 수십 년간 쓰이던 개념이다.

### 핵심 수식

```
연속 시간 시스템:
  h'(t) = A·h(t) + B·u(t)   ← 상태 업데이트
  y(t)  = C·h(t)             ← 출력

여기서:
  u(t) = 입력 (로봇의 센서 값)
  h(t) = 내부 상태 (시스템이 기억하는 것)
  y(t) = 출력 (로봇의 행동)
  A, B, C = 학습 가능한 파라미터
```

### 직관적으로 이해하기

```
비유: 물탱크 시스템

  물 유입량 u(t)
        ↓
  [탱크 수위 h(t)]  ← 현재 상태 (기억)
        ↓
  물 배출량 y(t)

수위는 이전 수위 + 유입량에 의해 결정됨
→ 시간의 흐름이 자연스럽게 반영됨
```

### 트랜스포머 vs SSM 비교

| | 트랜스포머 | SSM |
|---|---|---|
| 시간 처리 | 위치 인덱스로만 구분 | 연속적인 다이나믹스 모델링 |
| 메모리 | O(n²) — 모든 위치 참조 | O(n) — 내부 상태만 유지 |
| Temporal Structure | 이해 못 함 | 자연스럽게 포함됨 |
| 장점 | 표현력 강함 | 메모리 효율, 연속 다이나믹스 |

## 3. 이산화 (Discretization) — 컴퓨터에서 쓰려면

실제 컴퓨터는 연속 시간을 다룰 수 없다. 이산화가 필요하다.

```
연속 시간 → 이산 시간 변환:

  h_t = Ā·h_{t-1} + B̄·u_t
  y_t = C·h_t

  Ā = exp(∆·A)
  B̄ = (∆A)^{-1}·(exp(∆A) - I)·∆B
```

### Step Size (∆)의 의미 — 핵심!

∆는 **연속 시간을 얼마나 촘촘하게 샘플링할지** 결정한다.

```
∆가 작다 (예: 0.01초)
  → 시간을 촘촘하게 봄
  → 세밀한 변화를 포착
  → 정밀한(Fine) 동작 생성

∆가 크다 (예: 1초)
  → 시간을 듬성듬성 봄
  → 큰 변화만 포착
  → 거친(Coarse) 동작 생성
```

이게 DiSPo에서 **정밀도를 조절하는 핵심 메커니즘**이다.

## 4. S4 — 딥러닝에 SSM을 적용한 첫 시도

2022년 Gu et al.이 SSM을 딥러닝에 적용한 S4 모델을 발표했다.

### S4의 특징
- 긴 시퀀스를 효율적으로 처리
- Step Size ∆가 **고정값** — 입력에 관계없이 항상 같은 시간 간격

### S4의 한계
```
어떤 입력이 와도:
  "이 부분은 중요하니까 더 촘촘하게 봐야겠다" → 불가능
  항상 같은 ∆로 처리
```

---

## 5. Mamba — S4의 한계를 극복

2023년 Gu & Dao가 발표한 Mamba는 S4의 핵심 개선판이다.

### 핵심 혁신: 입력 의존적 파라미터 (Selective SSM)

```
S4:    B, C, ∆ = 고정값
Mamba: B_t = f_B(u_t)    ← 입력에 따라 동적 변화
       C_t = f_C(u_t)
       ∆_t = SoftPlus(f_∆(u_t))  ← 핵심!
```

입력을 보고 "이 순간은 중요하다 → ∆를 작게" 또는 "이 순간은 덜 중요하다 → ∆를 크게"를 **스스로 판단**한다.

### 직관적 비유
```
독서할 때:
  중요한 부분 → 천천히 꼼꼼하게 읽음 (∆ 작음)
  덜 중요한 부분 → 빠르게 훑음 (∆ 큼)

Mamba는 이걸 자동으로 한다
트랜스포머는 모든 부분을 동등하게 처리
```

## 6. DiSPo에서 Mamba를 쓴 이유

DiSPo는 Mamba의 Step Size를 한 단계 더 발전시켰다.

### DiSPo의 Step-Scaled Mamba

```
Mamba:  ∆_t = SoftPlus(f_∆(u_t))
               ↑ 모델이 스스로 결정

DiSPo:  ∆_t = r_t × SoftPlus(f_∆(u_t))
               ↑       ↑
          사용자 입력  모델 판단
```

r_t = Step-Scale Factor (사용자가 직접 지정)

```
r_t = 0.5  →  ∆가 절반으로  →  2배 정밀한 동작
r_t = 2.0  →  ∆가 2배로    →  2배 거친 동작
r_t = 1.0  →  원래 그대로
```

### 왜 이게 중요한가

```
기존: 정밀한 동작을 원하면 → 정밀한 데이터 새로 수집 필요
DiSPo: 거친 데이터만 있어도 → r_t 조절로 정밀한 동작 생성
```

데이터 수집 비용을 획기적으로 줄일 수 있다.

## 7. SSM 계보 정리

```
제어공학 SSM (수십 년 전)
  연속 다이나믹스 모델링의 기반 개념
        ↓
S4 (2022, Gu et al.)
  SSM을 딥러닝에 적용
  고정된 Step Size
        ↓
Mamba (2023, Gu & Dao)
  입력 의존적 파라미터
  동적 Step Size
  트랜스포머의 강력한 대안으로 주목
        ↓
DiSPo (2024/2026, RIRO Lab)
  Mamba에 사용자 제어 가능한 Step-Scale Factor 추가
  로봇 제어의 정밀도 문제에 적용
```

## 8. 트랜스포머 vs Mamba — 로봇 제어 관점 최종 비교

| 관점 | 트랜스포머 | Mamba |
|------|-----------|-------|
| 시간 이해 | 위치 인덱스 | 연속 다이나믹스 |
| 정밀도 조절 | 불가 | Step Size로 조절 가능 |
| 메모리 | 시퀀스 길이² | 시퀀스 길이 |
| 실시간 처리 | 느림 | 빠름 |
| 표현력 | 매우 강함 | 강함 |
| 로봇 제어 | 패턴 모방 수준 | 다이나믹스 이해 가능 |

---

## 9. 배운 점

SSM을 공부하면서 가장 인상 깊었던 것은 **Step Size ∆의 의미**였다.

제어공학에서 오래된 개념인 이산화(Discretization)의 파라미터 ∆가, Mamba에서는 "어느 순간에 집중할지"를 결정하는 attention 메커니즘처럼 동작하고, DiSPo에서는 "얼마나 정밀하게 움직일지"를 제어하는 도구가 된다.

같은 수식이 맥락에 따라 완전히 다른 의미를 갖는다는 게 흥미로웠다.
